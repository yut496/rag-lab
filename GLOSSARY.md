# 用語集（GLOSSARY）

このリポジトリで扱う RAG 関連の用語をまとめる。特定の実装・実験には依存しない一般的な定義。

| 用語 | 意味 |
|---|---|
| **RAG** (Retrieval-Augmented Generation) | 検索で集めた文脈を LLM に渡して回答させる構成。モデル内部の知識ではなく外部文書を根拠にする。 |
| **LLM** (Large Language Model) | 大規模言語モデル。回答テキストの生成を担う。 |
| **ingestion（取り込み）** | データ源（PDF / HTML / DB / テキスト等）から生テキストを読み込む工程。 |
| **chunking（チャンク分割）** | 文書を検索単位（チャンク）に割る工程。分割が雑だと正解が複数チャンクに割れる・文の途中で切れる、といった事故が起きる。 |
| **チャンク (chunk)** | 検索単位となる文字列の断片。どの文書由来かを示す出典とともに扱うことが多い。 |
| **チャンクサイズ / オーバーラップ** | 1チャンクの長さ／隣り合うチャンクの重なり。重なりは境界での文脈断裂を防ぐ。 |
| **embedding（埋め込み、エンべディング）** | テキストを固定次元のベクトルに変換すること。「座標を計算する」＝「固定された意味空間にテキストを配置する」と捉えられる（住所→緯度経度のジオコーディングの例え）。 |
| **ベクトル / 次元** | 埋め込みの出力。多数の float がまとまって1つの「意味の座標」を表す。入力の文字数が違っても必ず同じ次元に落ちる。 |
| **意味空間** | 埋め込みモデルが内部に固定的に持つ「意味の地図」。文書も問いも同じモデルで同じ空間に置くから距離比較が成立する。モデルが違えば空間も違う（索引を作り直す必要がある）。 |
| **非対称埋め込み / input_type** | 同じテキストでも「文書として」と「問いとして」で埋め方を変える最適化。文書側は document、問い側は query として埋める。 |
| **index（索引）** | 検索対象を保持する集合。本質は「テキスト・出典・ベクトル」を束ねたリスト。 |
| **retrieval（検索）** | 問いを埋め込み、索引の中から意味的に近いチャンクを引く工程。母集団全体ではなく上位数件だけを見る。 |
| **近傍検索 (nearest neighbor search)** | 意味空間内で問いベクトルに近い点を探す操作。retrieval の本体。 |
| **コサイン類似度 (cosine similarity)** | 2つのベクトルの向きの近さ（内積 ÷ ノルムの積）。retrieval で近さを測る代表的な尺度。 |
| **TOP_K** | retrieval で上位何件を文脈として採用するか。 |
| **context（文脈）** | retrieval で引いたチャンクをまとめ、LLM に実際に渡す文字列。出典ラベルを付けることが多い。 |
| **generation（生成）** | 文脈と問いを LLM に渡して回答を作らせる工程。 |
| **grounding（接地・捏造抑制）** | 回答を渡した文脈だけに根拠づけ、文脈に無いことは「分からない」と言わせること。 |
| **出典 (source)** | チャンクがどの文書から来たかを示すラベル。回答の根拠を辿るために保持する。 |
| **コーパス (corpus)** | 検索対象となる文書群。 |
| **embeddings API / 生成 API** | 埋め込みを返すサービスと、テキスト生成を返すサービス。両者は別モデル・別サービスのことがある。 |
| **RPM / TPM** | Requests / Tokens Per Minute。APIのレート上限。超過すると 429（Too Many Requests）が返り、待機して再試行する。 |
| **hallucination（ハルシネーション）** | 文脈に無い内容をもっともらしく捏造する現象。grounding で抑制を狙う。 |
| **faithfulness（忠実性）** | 回答が与えた文脈にどれだけ忠実かを測る評価観点。 |
| **precision / recall** | 検索・分類の評価指標。適合率（引いたもののうち正しい割合）／再現率（正解のうち引けた割合）。 |
| **lost in the middle** | 長い文脈の中ほどに置かれた情報を LLM が見落としやすい現象。→ [詳細](glossary/lost-in-the-middle.md) |
| **reranking / cross-encoder** | retrieval 後に候補を並べ替えて精度を上げる手法／そのためのモデル。 |
| **hybrid retrieval (dense / sparse)** | ベクトル検索 (dense) とキーワード検索 (sparse) を併用する検索方式。 |
| **BM25** | 単語頻度ベースの古典的なスパース検索スコアリング。 |
| **query rewriting** | 検索前に問いを書き換えてヒット率を上げる手法。 |
| **ANN / HNSW / IVF / FAISS** | 近似最近傍探索 (Approximate Nearest Neighbor) と、その代表的アルゴリズム・ライブラリ。総当たり検索を高速化する。 |
| **DRM** (Document-Level Retrieval Mismatch) | チャンク単位の検索が、文書レベルの大域的文脈を取りこぼす不整合。 |
| **SAC** (Summary-Augmented Chunking) | 各チャンクを文書レベルの要約で拡張し、DRM を軽減する手法。 |
