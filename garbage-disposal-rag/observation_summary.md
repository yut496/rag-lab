## 6. 観測の所見（改善候補・本フェーズでは一切直さない）

棄却 2/7・category一致 4/8 と低いが **直さず観測として記録**する。以下は所見（Phase 5 で検討）:

1. **SCORE_FLOOR(0.78) が一度も発火していない**: 非route棄却の全問が top_score 0.83〜0.92 で常に閾値超え。
   特に not_in_corpus の id8(0.887)/id9(0.918) は「粗大ごみ」トピックに意味的に近く高スコア
   ＝「トピックは在庫だが具体的な回答は不在」を**スコア閾値では弁別できない**（閾値方式の天井問題）。
2. **実質的な棄却は生成層(LLM)で起きている**: route/SCORE_FLOOR を素通りした問いでも、プロンプトの
   groundedness 指示により LLM が回答本文で「区の案内では確認できませんでした」と自己抑制
   （id3/id7/id8/id9/id11/id12/id14）。**システム指標 `abstained=false` でも実回答は棄却的**。
   → Phase 5 の grounded_ok / abstain_ok は、システム棄却フラグだけでなく回答本文の自己抑制も見るべき。
3. **正解チャンクが top_k 内・top-1 外**（id4:「粗大ごみとは（分解しても粗大ごみ）」が #3 ／
   id5:「資源・飲食用かん（菓子や海苔の缶も回収）」が #2、top-1 とは 0.003〜0.008 の僅差）。
   → top_category(top-1) 一致は retrieval を過小評価。**「expected_category ∈ top_k」**を retrieval_ok の定義候補に。
4. **ナビ/関連リンク・チャンクの上位混入**（id1/id6/id7/id8/id14 の「関連リンク」「家庭から出る資源とごみ」）:
   Phase 2 Hotfix で**既知・未対処**としたナビ混入が retrieval を直接汚染（id7 は 0.8803 のナビ3連が top2-4）。
   実害が観測された。→ ナビ除去（Phase 2 残課題）が retrieval 改善の第一候補。
5. **route() の取りこぼし**: 入口で拾えたのは id13(何ごみ)/id15(収集日)のみ。too_broad(id1)、
   言い回しが正規表現に非該当の item_classify（id3「不燃ごみですか」/ id12「燃えるゴミでいい？」）、
   not_in_corpus（id8,id9）は全て素通り。→ 正規表現routeは限定的。意味ベースの棄却判定が要るかも。
6. **id14（安全上最重要）**: top_k に「区で収集できないもの」は無い（top_category=不燃ごみ）。
   ただし **#3「プラスチック/分別不適物」チャンクに正解情報（モバイルバッテリーは発火危険で絶対に出さない＋
   小型充電式電池ルート）が在り**、LLM もそれを引用して「プラスチックの日に出せない」と回答した。
   正解情報が複数カテゴリ（不燃ごみ・電池 / プラ・分別不適物 / 小型充電式電池ページ＝**7ページ corpus 外**）に
   散在し、expected_category=区で収集できないもの の単一ラベルでは捉えにくい。「正しいチャンクが希少」を実証。
   → 安全系品目は専用ルート or expected_category ラベルの見直しが要検討。
